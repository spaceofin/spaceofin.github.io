ts 파일 실행하기

로컬 데이터베이스인 SQLite를 이용하다 보면 가끔 db 내의 데이터를 모두 삭제해야 할 필요가 있을 때가 있습니다.

이럴 때 db 내 데이터를 삭제하는 코드를 담은 .ts 파일을 생성하여 실행하는 방식으로 하는데, 새로 생성한 nextjs 프로젝트에서 .ts 파일을 실행하려 하자 여러 가지 문제가 발생했습니다.

아래는 문제가 된 파일입니다.

// deleteALLData.ts
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

async function deleteAllData() {
  try {
    await prisma.memo.deleteMany();
    console.log("All memo data deleted");
  } catch (error) {
    console.error("Error deleting data: ", error);
  }
}

deleteAllData();

ts 확장자 파일 실행하기와 Trouble 발생

node 명령어로는 .js 확장자 파일만 실행이 가능합니다. .ts 확장자 파일을 실행하기 위해서는 ts-node 패키지의 추가 설치가 필요합니다.

npm install ts-node --save-dev

ts-node 설치 후 파일 실행 명령어를 입력합니다.

npx ts-node src/app/scripts/deleteAllData.ts

하지만 이번엔 아래와 같은 에러입니다.

[not found image] [not found image]

(node:22424) Warning: To load an ES module, set "type": "module" in the package.json or use the .mjs extension.
(Use `node --trace-warnings ...` to show where the warning was created)
...
SyntaxError: Cannot use import statement outside a module

코드에서 import 부분이 문제라고 합니다.

해결 방법1. require 사용

import 대신 CommonJS의 require 구문을 사용하는 것으로 코드를 수정합니다.

(수정 전)

import { PrismaClient } from "@prisma/client";

(수정 후)

const { PrismaClient } = require("@prisma/client");

코드 수정 후 npx ts-node 명령어를 통해 .ts 파일을 실행해보면 정상작동합니다.

해결 방법2. tsx 사용

ts-node 대신 tsx를 사용하는 방법입니다.

npm install tsx --save-dev

tsx 명령어 설치 후 아래의 명령어로 .ts 파일을 실행해보니 잘 작동합니다.

npx tsx src/app/scripts/deleteAllData.ts

TypeError: Unknown file extension ".ts"
