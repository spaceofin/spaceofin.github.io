tailwindcss 동적 클래스 이름(dynamic class name) 사용하기

Trouble: tailwindcss에서 템플릿 리터럴(template literal) 사용이 안 돼요

tailwindcss에서 당연히 가능할 것 같았던 문법이 제대로 동작하지 않습니다. 아래와 같은 템플릿 리터럴 부분입니다.

const bgColor = `bg-${color}-300`;

문제가 되는 부분의 코드는 아래와 같습니다.

{dayTodos.map((todo, index) => {
	...
	const topClass = `top-${4 + todo.index * 4}`;
	const bgColor = `bg-${todo.color}-300`;
	return (
		<div
		  key={index}
		  className={`absolute ${topClass} left-0 w-full h-4 z-0 ${bgColor}`}></div>
	);
	})}

공식 문서를 찾아보면 Don’t use props to build class names dynamically라고 안내하고 있습니다.

class names을 동적으로 사용하면 안되며 항상 완전한 class names를 사용하라고 합니다.

// error code
<div class="text-{{ error ? 'red' : 'green' }}-600"></div>

// error code
function Button({ color, children }) {
  return (
    <button className={`bg-${color}-600 hover:bg-${color}-500 ...`}>
      {children}
    </button>
  )
}

위와 같은 코드는 tailwindcss에서 작동하지 않습니다.

tailwindcss의 dynamic class name에서 허용된 것

1. 완전한 클래스 이름(complete class names) 사용

전체 class names을 조건에 따라 사용하는 것이 가능합니다.

<div class="{{ error ? 'text-red-600' : 'text-green-600' }}"></div>

2. 정적 클래스 이름(static class names) 를 맵핑(mapping)하여 사용

function Button({ color, children }) {
  const colorVariants = {
    blue: 'bg-blue-600 hover:bg-blue-500',
    red: 'bg-red-600 hover:bg-red-500',
  }

  return (
    <button className={`${colorVariants[color]} ...`}>
      {children}
    </button>
  )
}

3. safelist 사용하기

만일 bg-${color}-300와 같은 동적 클래스를 사용하고 싶다면 tailwind.config.js 파일에서 safelist 옵션을 추가하는 방법이 있습니다.

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    './pages/**/*.{html,js}',
    './components/**/*.{html,js}',
  ],
  safelist: [
    'text-2xl',
    'text-3xl',
    {
      pattern: /bg-(red|green|blue)-(100|200|300)/,
    },
  ],
  // ...
}

하지만 안전 목록(safelisting)은 마지막 수단으로 사용해야 하며, 특정 콘텐츠에서 클래스 이름을 스캔할 수 없는 상황에서만 사용해야 합니다. 이러한 상황은 드물며, 이 기능이 필요할 일은 거의 없다고 공식 홈페이지에서는 안내하고 있습니다.

Trouble 해결하기

문제가 되는 코드 부분을 수정합니다.

방법1. static class names mapping

const todoColors: Record<string, string> = {
  gray: "bg-gray-300",
  purple: "bg-purple-300",
  yellow: "bg-yellow-300",
  pink: "bg-pink-300",
  orange: "bg-orange-300",
  blue: "bg-blue-300",
  red: "bg-red-300",
  green: "bg-green-300",
};

const topClasses = ["top-4", "top-8", "top-12", "top-16", "top-20"];

{dayTodos.map((todo, index) => {
	...
	return (
		<div
		  key={index}
		  className={`absolute ${
			topClasses[todo.index]
		  } left-0 w-full h-4 z-0 ${todoColors[todo.color]}`}></div>
	);
})}

동적으로 사용되던 class names를 static class names로 바꾸어 사용하도록 하였습니다. 정상적으로 작동합니다.

방법2. safelist 사용

//tailwind.config.ts
import type { Config } from "tailwindcss";

const config: Config = {
  ...
  safelist: [
    {
      pattern:
        /bg-(gray|blue|orange|green|red|yellow|pink|purple|sky)-(200|300|400)/,
    },
    {
      pattern: /top-\d+/,
    },
  ],
  ...
};
export default config;

사용할 색들을 미리 지정해서 safelist의 패턴으로 추가하고 top-숫자 형태 역시 패턴으로 추가하였습니다.

{dayTodos.map((todo, index) => {
	const topClass = `top-${4 + todo.index * 4}`;
	const bgColor = `bg-${todo.color}-300`;
	return (
	  <div
	    key={index}
	    className={`absolute ${topClass} left-0 w-full h-4 z-0 ${bgColor}`}
	  ></div>
	);
})}

미리 패턴 지정을 해두었기 때문에 위와 같은 코드가 정상적으로 작동하였습니다.
