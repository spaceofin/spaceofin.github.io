memo editing 기능 구현하기

생성된 메모의 title이나 description을 수정 후 다시 db에 저장하는 기능을 구현하려고 합니다.

Dynamic routes를 이용한 memo edit 페이지 생성

각 메모별 별도의 edit 페이지를 가지게 하기 위해 dynamic route 를 이용합니다. 메모 수정 작업은 /memos/[id]/edit 페이지에서 수행되며, [id] 부분 즉, dynamic segments 부분은 build time에 채워넣어 집니다.

params prop은 Promise 입니다. 따라서 async/await 를 사용해야 합니다.

// memo-board/src/app/memos/[id]/edit/page.tsx

import { db } from "@/app/db";
import { notFound } from "next/navigation";
import MemoEditSection from "./memo-edit-section";

export default async function MemoEdit({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const memo = await db.memo.findFirst({
    where: { id: parseInt(id) },
  });
  if (!memo) return notFound();
  
  return <MemoEditSection memo={memo} />;
}

각 페이지마다 해당 메모 데이터를 prop으로 가지는 MemoEditSection 컴포넌트를 리턴합니다.

editMemo 서버 액션(server action) 구현

클라이언트 컴포넌트 내에서 호출하기 위해 server action으로 editMemo 함수를 구현합니다.

"use server";

import { db } from "../db";
import { redirect } from "next/navigation";

...

export async function editMemo(id: number, title: string, content: string) {
  await db.memo.update({
    where: { id },
    data: { title, content },
  });
  
  redirect("/");
}

클라이언트 컴포넌트 내 서버 액션 사용

실질적으로 메모 수정 작업을 진행할 MemoEditSection 클라이언트 컴포넌트입니다.

title과 content를 상태변수로 관리하며 초기값으로 기존 메모의 텍스트를 할당하고 수정을 마친 후 Done 버튼을 클릭하면 editMemo 서버 액션 실행을 포함하는 handleEditDoneClick 함수가 실행됩니다.

// memo-board/src/app/memos/[id]/edit/memo-edit-section.tsx
"use client";

import { editMemo } from "@/app/service/memosServies";
import { useState } from "react";

type memoProps = {
  id: number;
  title: string;
  content: string;
};

export default function MemoEditSection({ memo }: { memo: memoProps }) {
  const [title, setTitle] = useState(memo.title);
  const [content, setContent] = useState(memo.content);

    try {
      await editMemo(memo.id, title, content);
    } catch (error) {
      console.error("Error editing memo:", error);
    }
  };

	  ... 
	)
}
