CompileMDX로 frontmatter 데이터 분리하기

Trouble: CompileMDX 함수의 리턴 타입 에러

getMetadata 함수에 리턴 타입 지정을 하지 않았더니 frontmatter 데이터의 title을 출력했을 때 맞는 문자열들이 화면에 출력되기는 하지만 코드 자체에 빨간색으로 에러 표시가 생겼습니다.

async function getMetadata({ source }: { source: string }) {
  return await compileMDX({
    source,
    options: {
      parseFrontmatter: true,
    },
  });
}

그래서 mdx post들의 타입 지정을 하고 getMetadata 함수에 리턴 타입을 할당해 주었습니다.

export type MDXPost = {
  content: string;
  frontmatter: { title: string; date: string; tags: string[] };
};

async function getMetadata({
  source,
}: {
  source: string;
}): Promise<CompileMDXResult<MDXPost>> {
  return await compileMDX({
    source,
    options: {
      parseFrontmatter: true,
    },
  });
}

그런데 여전히 title 부분에 빨간색 에러 표시줄이 나타납니다.

게다가 post.frontmatter. 까지 적었을 때 다시 content와 frontmatter 옵션이 나타납니다.

Trouble 원인 찾기

getMDXPosts 함수로 반환받는 posts 배열을 출력해 보았습니다.

post 객체는 content과 frontmatter 속성을 가지고 있습니다. 그리고 frontmatter 속성은 다시 title, date, tags 속성을 가집니다. 문제가 없어 보입니다.

원인을 찾기 위해 compileMDX 의 definition을 확인해봅니다.

compileMDX는 아래와 같은 녀석입니다. CompileMDXResult는 아래와 같습니다.

여기에서 문제의 원인이 보입니다. compileMDX의 반환 타입에서 TFrontmatter는 CompileMDXReulst의 제네릭 매개변수(generic parameter) 이며 CompileMDXResult의 타입이 content와 frontmatter 속성으로 정의되어 있습니다.

즉, Promise<CompileMDXResult<MDXPost>> 에서 MDXPost 전체에 대한 타입 정의가 아닌 frontmatter에 대한 타입 정의만 넣어줘야 했던 것입니다.

Trouble 해결하기

코드를 수정합니다.

export type Frontmatter = {
  title: string;
  date: string;
  tags: string[];
};

async function getMetadata({
  source,
}: {
  source: string;
}): Promise<CompileMDXResult<Frontmatter>> {
  return await compileMDX({
    source,
    options: {
      parseFrontmatter: true,
    },
  });
}

빨간색 에러 표시줄이 사라졌으며, post.frontmatter. 까지 입력했을 때 맞는 속성들이 나타납니다.
