로컬 스토리지(local storage), 세션 스토리지(session storage)

웹 스토리지라고 하면 보통 로컬 스토리지(local storage)와 세션 스토리지(session storage)를 말합니다.

로컬 스토리지

로컬 스토리지(local storage)의 데이터는 클라이언트(브라우저)에만 저장되며 서버로 전송되지 않습니다. 저장 용량은 브라우저마다 다르나 약 5MB~10MB 정도이며 명시적으로 삭제하지 않으면 브라우저를 닫아도 데이터가 영구적으로 유지됩니다. 사용자 설정, 테마 등 애플리케이션 데이터를 저장하는 데 주로 사용되며 서버로 전송되지 않으므로 네트워크 트래픽에 영향을 끼치지 않습니다.

세션 스토리지

세션 스토리지(session storage)의 데이터는 로컬 스토리지와 마찬가지로 클라이언트(브라우저)에 저장되며 서버로 전송되지 않습니다. 저쟝 용량은 브라우저마다 다르나 약 5MB 정도입니다. 로컬 스토리지와의 차이점은브라우저의 탭 또는 창이 닫히면 데이터가 삭제된다는 점입니다.(데이터가 브라우저 세션 동안에만 유지됨). 따라서 브라우저 탭 단위로 데이터를 저장하여 탭 간 독립된 데이터를 관리하는 경우에 사용이 적합합니다.(예: 폼 데이터) 페이지 새로고침 시에도 데이터가 유지됩니다.

보안 측면에서의 로컬 스토리지와 세션 스토리지

로컬 스토리지와 세션 스토리지는 클라이언트 측에 데이터를 저장한다는 점에서 쉽게 접근 가능하므로, 민감한 정보(예: 비밀번호, 개인 정보)를 저장하지 않아야 하며 만약 민감한 정보를 저장해야 한다면 반드시 암호화를 적용하여야 합니다. 로컬 스토리지에 저장된 데이터는 브라우저의 JavaScript 코드에서 접근할 수 있으므로, XSS 공격에 취약할 수 있습니다. XSS 취약점을 제거하는 것이 필수적인데 이를 위해서 사용자 입력값을 적절히 검증하고, HTML 또는 JavaScript 인젝션을 방지하기 위한 방어 코드를 추가해야 합니다. 이를 위해 데이터를 HTML로 출력할 때 반드시 이스케이프 처리를 해야 합니다.
