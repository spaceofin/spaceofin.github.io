Event Bubbling, Capturing, preventDefault 그리고 stopPropagation

bubbling과 capturing

버블링(bubbling)과 캡쳐링(capturing)은 DOM에서 이벤트가 발생할 때 이벤트가 전파되는 방식에 대한 개념입니다.

버블링(bubbling)은 이벤트가 발생한 요소에서 시작하여 상위 요소들로 전파되는 것으로 이벤트 전파의 기본 동작입니다. 캡쳐링(capturing)은 버블링과 반대로, 이벤트가 상위 요소에서 시작하여 하위 요소로 전파되는 것으로 기본적으로 비활성화되어 있습니다.

bubbling

버블링에서 이벤트는 DOM 트리를 따라 위로 올라가면서 부모 요소들에 전파됩니다. 최종적으로는 최상위 요소(window)까지 전파됩니다.

<body>
	<div id="parent">
	  <button id="child">Click me</button>
	</div>
	<div id="sibling-of-parent"></div>
</body>

document.getElementById("child").addEventListener("click", function() {
  console.log("child clicked!");
});
document.getElementById("parent").addEventListener("click", function() {
  console.log("parent clicked!");
});
document.getElementById("sibling-of-parent").addEventListener("click", function() {
  console.log("sibling-of-parent clicked!");
});
document.body.addEventListener("click",function(){
	console.log("body clicked!");
})

위의 예제 코드에서 Click me 버튼을 클릭하면 콘솔에 출력되는 메시지

child clicked!
parent clicked!
body clicked!

클릭 이벤트가 발생한 버튼 요소와 그 부모 요소들에 이벤트가 전파되어 동작하는 것을 확인할 수 있습니다.

capturing

캡처링은 기본적으로 비활성화 되어 있으므로, 이벤트가 capture 단계에서 처리되도록 명시적으로 설정해야 합니다. (addEventListener 함수의 세 번째 옵션 인자로 { capture: true } 또는 true 값을 할당)

<body>
	<div id="parent">
	  <button id="child">Click me</button>
	</div>
	<div id="sibling-of-parent"></div>
</body>

document.getElementById("child").addEventListener("click", function() {
  console.log("child clicked!");
});
document.getElementById("parent").addEventListener("click", function() {
  console.log("parent clicked!");
}, { capture: true });
document.getElementById("sibling-of-parent").addEventListener("click", function() {
  console.log("sibling-of-parent clicked!");
});
document.body.addEventListener("click",function(){
	console.log("body clicked!");
})

위의 예제 코드에서 Click me 버튼을 클릭하면 콘솔에 출력되는 메시지

parent clicked!
child clicked!
body clicked!

클릭 이벤트가 캡쳐 단계에서 먼저 발생하고 하위 요소로 전파되어 동작하는 것을 확인할 수 있습니다.

preventDefault와 stopPropagation

event.preventDefault()

이벤트의 기본 동작을 방지합니다. 예를 들면, <a> 태그 클릭 시 페이지가 이동하는 기본 동작이나, <form> 태그에서 폼을 제출하고 페이지가 새로고침되는 기본 동작을 막고 싶을 때 사용합니다.

event.stopPropagation()

이벤트가 부모 또는 자식 요소로 전파되는 것을 방지합니다.(capturing, bubbling 전파 중단) 예를 들어, 자식 요소에서 클릭 이벤트를 처리하고 부모 요소에서 같은 이벤트가 발생하지 않도록 할 때 유용합니다.

<body>
	<div id="parent">
	  <button id="child">Click me</button>
	</div>
</body>

document.getElementById("child").addEventListener("click", function(event) {
  console.log("child clicked!");
  event.stopPropagation();
});
document.getElementById("parent").addEventListener("click", function() {
  event.stopPropagation();
  console.log("parent clicked!");
});
document.body.addEventListener("click",function(){
	console.log("body clicked!");
})

위의 예제 코드에서 Click me 버튼을 클릭하면 콘솔창에 출력되는 메시지

child clicked!
parent clicked!

(버블링 단계 중 parent 요소에서 이벤트 전파가 중단되었습니다.)
